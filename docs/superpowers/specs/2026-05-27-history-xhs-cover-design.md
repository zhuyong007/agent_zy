# 历史知识模块小红书封面方案设计

## 背景

历史知识模块当前会生成小红书图文策划，包括主题、摘要、图文卡片、生图提示词和正文。新增能力是在生成历史知识时同步生成一份“小红书风格封面”方案，用户可以在历史详情页复制封面文案和封面生图提示词，再交给外部生图工具使用。

本次不接入新的生图 API，不新增网络调用，不要求额外密钥。

## 目标

- 历史知识生成完成后，payload 中包含整篇笔记的封面方案。
- `/history` 详情页在“图文拆解”上方展示封面方案。
- 用户可以一键复制封面文案和封面生图提示词。
- 旧历史记录没有封面字段时仍能正常展示，不报错。

## 非目标

- 不在页面内直接生成图片。
- 不下载、保存或上传封面图片。
- 不修改小红书数据同步功能。
- 不重构历史模块整体布局。

## 数据结构

在 `HistoryPostPayload` 增加可选字段：

```ts
cover?: {
  title: string;
  subtitle: string;
  imageText: string;
  prompt: string;
};
```

字段保持可选，兼容旧通知记录和测试 fixtures。`title` 和 `subtitle` 用于页面展示与复制封面文案，`imageText` 表示封面上建议出现的中文文字，`prompt` 是外部生图工具使用的封面提示词。

## 生成逻辑

`agents/history-agent/src/index.ts` 的模型 JSON 输出要求增加 `cover` 字段。模型应围绕主题产出小红书首图封面方案，强调：

- 竖版首图封面。
- 强标题层级。
- 历史知识感与准确时代氛围。
- 中文文字留白和可读性。
- 适合小红书信息流的首屏吸引力。

如果模型未返回 `cover`，或返回值缺少必要字符串字段，后端使用 fallback 派生：

- `title` 使用当前主题。
- `subtitle` 基于摘要生成短副标题。
- `imageText` 组合主题、摘要和第一张卡片的图片文字。
- `prompt` 基于主题、摘要和第一张卡片提示词生成封面生图提示词。

这样可以保证新生成记录稳定带有封面方案，同时不要求模型输出完全可靠。

## 前端交互

在 `apps/web/src/components/history-page.tsx` 中，当选中历史记录存在时：

- 在“图文拆解”之前新增“封面方案”区块。
- 展示封面标题、副标题、封面文字建议和封面生图提示词。
- 提供“复制封面文案”和“复制封面提示词”两个按钮。
- 如果旧 payload 没有 `cover`，前端可以隐藏封面区块，页面继续展示原有图文拆解和正文。

样式只新增局部 class，放在 `apps/web/src/styles.css` 的历史模块样式区域，保持现有历史页视觉语言。

## 测试

- 更新 `agents/history-agent` 测试，覆盖模型返回 cover 和 fallback cover。
- 更新 control-plane 生成历史的测试 fixture，确认通知 payload 带 cover。
- 更新 `HistoryPage` 测试，确认封面区块展示且复制按钮写入正确文本。
- 旧 payload 兼容通过现有页面测试和可选字段类型保证。

## 风险与处理

- 模型输出字段不稳定：用后端 fallback 兜底。
- 旧通知没有 cover：字段可选，前端隐藏封面区块。
- 提示词过长或过短：复用现有历史 agent 的校验和规范化思路，必要时对 fallback prompt 控制长度。
- 工作区已有未提交改动：实现时只做最小范围叠加，不回滚用户已有修改。
