# 梦幻西游角色资产类型设计

## 目标

在“梦幻西游交易”模块中，把“角色”增加为与“召唤兽”“装备”并列的资产交易类型，并把过长的页签名称“角色召唤兽及装备交易记录”缩短为“资产交易记录”。

## 命名与界面

- 页签名称统一为“资产交易记录”，与相邻的“跨服交易记录”“物价记录”保持同一命名层级。
- 页内主标题改为“角色 / 召唤兽 / 装备人民币盈亏”，明确“资产”包含的范围。
- 资产类型下拉框按“角色、召唤兽、装备”展示，对应数据值为 `role`、`summon`、`equipment`。
- 资产列表和空状态使用“资产”作为统称，不再只写“召唤兽或装备”。
- 对召唤兽和装备，现有 `characterName` 字段在界面中改称“归属角色”，避免与资产类型“角色”混淆。
- 当资产类型为“角色”时，`name` 表示交易角色名称，保留“区服”，隐藏不适用的“归属角色”输入框；切换为角色类型或提交角色记录时不保存该字段的陈旧输入值。

## 数据与兼容性

- 扩展共享类型 `MhxyAssetFlipCategory`，加入 `role`。
- API 校验和服务层规范化逻辑接受 `role`，并把相关错误文案改为通用的“资产记录”或明确列出三种资产类型。
- 现有 `summon`、`equipment` 数据和值保持不变，不需要迁移。
- 角色资产沿用现有人民币买入价、卖出价、持有/售出状态和盈亏计算规则，不引入角色专属字段。
- 数据同步、导入和导出继续复用 `assetFlips` 集合；新增的 `role` 值必须能够通过全量数据校验和回放。

## 验证

- 共享类型和后端校验允许 `role`，仍拒绝未知资产类型。
- 服务测试覆盖角色资产的创建、更新、汇总和数据集导入。
- 页面测试覆盖新页签名称、角色类型选项、角色标签展示，以及角色类型下“归属角色”字段的隐藏与清理。
- 现有召唤兽、装备记录的显示、编辑和盈亏行为保持不变。
- 运行相关单元测试、TypeScript 类型检查和 Web 构建。

## 非目标

- 不增加门派、等级、修炼等角色专属属性。
- 不新增按资产类型筛选、分组或独立汇总。
- 不改变跨服交易记录和物价记录的业务逻辑。
