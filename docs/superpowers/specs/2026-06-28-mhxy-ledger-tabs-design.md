# 梦幻西游跨服与角色资产视觉优化设计

## 跨服交易：调度台

单一目标是快速判断“现在持有什么、值多少、准备转到哪里”。交易与库存转移录入默认收起，库存与资金指标进入首屏。视觉延续行情终端的深海墨蓝 `#0C1727`、雾白 `#EDF7F4`、数据灰 `#92A5BA`，新增路线蓝 `#65B5FF` 表示跨服方向，行情青 `#42D6B0` 表示正向价值。

结构：

```text
[跨服调度台 + 记录交易 + 库存转移]
[库存成本 | 市场估值 | 浮盈 | 已实现收益]
[当前库存：道具 / 数量 / 成本 / 估值 / 目标区服]
[交易流水] [转移轨迹]
```

记忆点是库存行中的“源区服 → 目标区服”路线表达，结构本身传达跨服调度，而不是装饰。

## 角色召唤兽及装备：资产簿

单一目标是区分“还持有什么”和“已经赚亏多少”。新增/编辑表单收进右上角折叠入口，当前库存与交易历史成为两张连续账页。继续使用行情青表示持有，琥珀 `#F2B35D` 表示已售与亏损提醒。

结构：

```text
[角色资产簿 + 添加资产]
[库存价值 | 当前库存 | 总盈亏 | 已售出]
[持有账页：资产 / 成本 / 状态 / 口径]
---------------- 双账页分隔 ----------------
[已售账页：资产 / 买入 / 卖出 / 盈亏]
```

## 交互与适配

- 所有录入入口使用原生 `details/summary`，默认关闭并支持键盘。
- 编辑现有记录时自动展开对应录入入口。
- 桌面端维持高密度表格，窄屏改为卡片式行布局。
- 动效只用于页面进入和折叠入口反馈，并尊重 reduced-motion。
- 不修改 API、数据模型与会计计算口径。

## 测试

- 验证跨服交易与库存转移表单位于默认关闭的次要入口。
- 验证跨服库存和两类流水始终存在。
- 验证角色资产表单位于默认关闭的次要入口。
- 验证角色当前库存和已售历史仍按状态隔离。
