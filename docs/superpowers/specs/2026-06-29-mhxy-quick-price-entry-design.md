# 梦幻西游当前道具快捷新增物价记录设计

## 目标

用户查看某个道具的价格走势时，可以直接为当前道具追加一期价格，不再重复填写道具名和来源。

快捷新增只减少重复输入，不改变价格快照的数据模型、聚合方式或估值逻辑。

## 第一性原理

一条新价格快照包含两类信息：

- 已由当前趋势确定的上下文：道具名、来源 / 区服。
- 本次观察产生的新信息：货币形式、价格、采集时间；使用游戏币时还包括当时兑换比例。

因此快捷表单必须继承并锁定当前趋势的道具名和来源，只让用户填写本次发生变化的信息。页面顶部原有“添加记录”继续承担创建全新道具或新来源序列的职责。

## 交互设计

- 在当前道具趋势标题区增加“＋记录新价格”入口。
- 点击入口后，在趋势标题附近展开轻量表单。
- 表单顶部只读展示“道具名 · 来源 / 区服”，让用户确认记录归属。
- 默认货币形式为人民币，采集时间为当前本地时间。
- 人民币模式只填写人民币单价。
- 游戏币模式填写游戏币单价（万），并要求填写当时兑换比例（每 1 万游戏币折合人民币）。
- 提交期间禁用保存按钮，避免重复提交。
- 提交成功后关闭并重置快捷表单，刷新价格数据，当前道具保持选中。
- 提交失败后保持表单打开并保留输入，通过页面现有错误区域展示失败原因。
- 页面顶部“＋添加记录”及完整表单保持不变，继续用于新建其他道具或来源。

## 组件与数据流

`PriceTrendWorkspace` 继续负责当前价格序列的选择和展示，并向快捷表单传入当前序列的 `itemName` 与 `sourceName`。

快捷表单提交时组装现有 `MhxyPriceSnapshotInput`：

- `itemName` 固定取当前序列道具名。
- `serverName` 固定取当前序列来源名。
- `currency`、价格字段和 `capturedAt` 取用户输入。
- 游戏币模式附带 `gameCoinUnitPriceWan` 与 `rmbPerGameCoinWan`。

表单复用现有 `createMhxyPriceSnapshot` mutation，不新增 API 或后端字段。刷新后的序列键仍由“来源 + 道具名”组成，因此当前选择可保持不变。

## 状态与错误处理

- 快捷表单的展开状态属于当前趋势工作区。
- 切换观察道具时关闭并重置已展开的快捷表单，防止旧输入被误提交到新上下文。
- API 成功后关闭并重置表单。
- API 失败时不重置原生表单，用户可以修改后重试。
- 当没有任何价格记录时不显示快捷入口，只保留现有空状态中的完整添加入口。

## 视觉与响应式

- 快捷入口使用现有行情青色和行情终端视觉语言，层级高于顶部的通用“添加记录”。
- 表单采用紧凑布局，不遮挡当前道具名称和最新价。
- 窄屏下表单改为单列并占满趋势卡片可用宽度。
- 输入框保持清晰标签和键盘焦点；只读上下文使用可读文本而非禁用输入框。

## 测试与验收

组件测试覆盖：

- 当前趋势显示快捷新增入口，空状态不显示。
- 快捷表单只显示本次需要输入的字段，并展示当前道具与来源。
- 人民币模式提交时自动继承当前道具名和来源。
- 游戏币模式显示并提交必填兑换比例。
- 切换观察道具后，快捷表单使用新道具上下文且不保留旧输入。
- 提交失败时保留输入和展开状态。
- 提交成功时关闭表单。
- 原有完整“添加记录”入口继续存在。

验收标准：用户从查看当前道具走势到追加一条人民币价格，只需点击快捷入口，填写价格并保存；无需再次输入道具名或来源。
