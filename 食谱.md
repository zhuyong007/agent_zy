你是一名资深全栈工程师。请在我现有的个人 Agent 助手项目中，在「工具」模块下新增一个「孩子食谱」工具。

这个工具的目标不是简单生成几道菜，而是根据孩子出生日期、当前月龄、近期饮食记录、备注信息、季节食材和营养均衡原则，持续规划适合孩子的每日辅食 / 正餐食谱，并记录孩子每天吃过什么，用长期记忆降低重复度，合理安排未来食谱。

一、模块基本信息

模块名称：child-meal-planner

中文名称：孩子食谱

入口位置：
新增到现有项目的「工具」模块中，作为一个独立工具入口。

核心目标：

1. 根据孩子出生日期自动计算当前月龄。
2. 按月龄推荐适合的食谱。
3. 食谱要营养均衡，有主食、蔬菜、肉类、蛋类、水果。
4. 食材尽量选择当季常见食材。
5. 记录每天吃过的辅食 / 正餐。
6. 根据历史记录规划未来食谱，减少重复。
7. 支持填写身高、体重、睡眠、喝奶量等备注信息。
8. 不推荐药物、保健品、营养补剂、具体商品。
9. 只做日常饮食建议，不做医疗诊断。

二、默认孩子信息

请内置一个默认孩子档案：

* 出生日期：2025-01-22
* 系统需要根据当前日期自动计算：

  * 年龄
  * 月龄
  * 当前所处喂养阶段

注意：
不要把月龄写死。
每次打开工具或生成食谱时，都应该根据当前日期重新计算月龄。

三、用户可填写信息

页面中需要有一个「孩子信息」区域。

必填 / 默认信息：

* 出生日期：默认 2025-01-22，可修改
* 当前月龄：系统自动计算，不允许手动乱填，但可以展示

选填信息：

* 身高
* 体重
* 是否早产
* 是否有过敏食物
* 是否有不爱吃的食物
* 是否有近期便秘 / 腹泻 / 胃口差等备注
* 每日喝奶量，例如：最近每天喝 185ml 鲜奶
* 每日睡眠时间
* 每天几点醒
* 每天几点睡
* 午睡时间
* 当前咀嚼能力，例如：泥糊、碎末、小块、手指食物、接近成人软饭
* 家庭常备食材
* 家庭忌口
* 做饭设备，例如：蒸锅、炒锅、电饭煲、空气炸锅

页面需要有提示栏：

“填写身高、体重、喝奶量、睡眠、过敏史和咀嚼能力后，系统可以更准确地安排食谱。未填写时，将按当前月龄的一般饮食原则推荐。”

四、备注信息功能

需要提供一个「近期备注」区域，用来记录当前孩子的生活和饮食状态。

示例备注：

* 最近每天喝 185ml 鲜奶
* 早上 7:30 醒，晚上 21:00 睡
* 午睡 1.5 小时
* 最近不太爱吃青菜
* 最近喜欢吃鸡蛋羹
* 最近有点便秘
* 今天已经吃过香蕉
* 本周吃过虾，未发现不适

备注要求：

1. 用户可以随时新增备注。
2. 备注需要保存到长期存储中。
3. 生成食谱时需要读取最近 7 天、14 天、30 天的备注作为参考。
4. 备注不应该覆盖历史饮食记录，而是作为辅助上下文。
5. 备注里如果出现疾病、严重过敏、持续腹泻、呕吐、体重异常等情况，只能提示咨询儿科医生，不能给药物或医疗方案。

五、每日饮食记录功能

需要提供「今天吃了什么」记录功能。

用户可以记录：

* 日期
* 餐次：早餐 / 午餐 / 晚餐 / 加餐 / 奶 / 水果
* 食物名称
* 食材
* 烹饪方式
* 份量，可选
* 孩子接受度：喜欢 / 一般 / 不喜欢 / 拒绝
* 是否过敏或不适
* 备注

示例：

日期：2026-06-11
餐次：午餐
食物名称：番茄牛肉碎软饭 + 蒸南瓜
食材：番茄、牛肉、米饭、南瓜
烹饪方式：炒 + 蒸
接受度：喜欢
备注：牛肉吃得不错，南瓜吃了一半

记录要求：

1. 每天吃过的辅食 / 正餐必须可以长期保存。
2. 生成未来食谱时，要读取最近 7 天和 30 天饮食记录。
3. 不要连续多天推荐同一主菜。
4. 不要短期内重复同一种肉类、同一种蔬菜组合。
5. 如果某种食物被标记为过敏或不适，后续推荐必须避开，并提示谨慎。
6. 如果某种食物连续被拒绝，不要频繁重复推荐，可以换做同类营养替代食材。

六、食谱生成规则

用户可以生成：

* 今日食谱
* 明日食谱
* 未来 3 天食谱
* 未来 7 天食谱

每一天需要包含：

* 早餐
* 午餐
* 晚餐
* 加餐 / 水果
* 喝奶 / 饮水备注，可选
* 今日营养重点
* 今日烹饪安排
* 今日避免重复说明

每餐建议包含：

* 食物名称
* 主要食材
* 烹饪方式
* 简单做法
* 适合月龄的处理方式
* 营养目的
* 注意事项

营养均衡要求：

1. 每天要尽量覆盖：

   * 主食 / 谷物
   * 蔬菜
   * 肉 / 鱼 / 虾 / 鸡 / 猪 / 牛等动物性食物
   * 蛋类
   * 水果
   * 奶类或奶制品，根据用户备注判断
2. 有素有肉有蛋有水果。
3. 食材尽量当季、常见、容易购买。
4. 不要推荐复杂难做的菜。
5. 不要推荐高盐、高糖、重油、辛辣、刺激性食物。
6. 不要推荐蜂蜜、整颗坚果、容易噎住的大块硬食物。
7. 不要推荐药物、保健品、补剂或具体商品。
8. 如涉及过敏高风险食材，需要提示“首次少量尝试，观察反应”，但不要过度医疗化。
9. 如果孩子不足对应食材适用月龄，必须避免推荐。

七、烹饪便利性要求

用户明确希望做饭方便。

如果一餐需要准备两个菜，优先设计为：

* 一个蒸菜 + 一个炒菜
* 一个蒸菜 + 一个煮粥 / 软饭
* 一个炖煮类 + 一个水果
* 一个主菜 + 一个简单配菜

示例：

午餐：

* 番茄牛肉碎软饭：炒
* 蒸胡萝卜山药条：蒸

这样可以让用户一边蒸、一边炒，节省时间。

系统生成食谱时需要额外输出「做饭顺序建议」，例如：

1. 先把南瓜和鸡蛋羹放入蒸锅。
2. 蒸的同时处理番茄和牛肉碎。
3. 最后炒番茄牛肉，拌入软饭。

八、季节食材规则

系统需要根据当前日期判断季节。

可按中国北方家庭常见季节食材粗略划分：

春季：
菠菜、油菜、莴笋、芦笋、豌豆、草莓、梨、苹果、鸡肉、鱼肉、鸡蛋

夏季：
西红柿、黄瓜、丝瓜、冬瓜、茄子、西葫芦、玉米、毛豆、桃、西瓜、蓝莓、鸡肉、鱼肉、虾、鸡蛋

秋季：
南瓜、山药、莲藕、胡萝卜、白萝卜、苹果、梨、葡萄、牛肉、猪肉、鸡蛋、鱼肉

冬季：
白菜、娃娃菜、土豆、胡萝卜、白萝卜、南瓜、山药、苹果、橙子、牛肉、猪肉、鸡肉、鸡蛋

要求：

1. 食谱优先使用当季食材。
2. 可以使用全年常见食材，例如鸡蛋、米饭、面条、猪肉、鸡肉、胡萝卜、土豆等。
3. 允许用户自定义所在地区，后续可扩展更准确的季节食材库。
4. 如果没有地区信息，默认按中国北方家庭常见食材处理。
5. 季节食材数据需要独立配置，不要硬编码在组件里。

九、重复度控制

系统需要实现食谱去重逻辑。

生成食谱时需要参考：

* 最近 3 天吃过什么
* 最近 7 天吃过什么
* 最近 30 天高频食材
* 被孩子拒绝的食材
* 被标记过敏 / 不适的食材
* 最近已经安排过的未来食谱

去重规则：

1. 3 天内不要重复同一道主菜。
2. 7 天内不要重复完全相同的食材组合。
3. 7 天内动物蛋白尽量轮换，例如鸡肉、猪肉、牛肉、鱼、虾、蛋。
4. 蔬菜颜色尽量轮换，例如绿色叶菜、橙色根茎、瓜茄类。
5. 水果不要每天都只推荐同一种。
6. 如果孩子喜欢某种食物，可以适度出现，但不要每天重复。
7. 如果某种食材刚被拒绝，至少间隔 3 天后再以不同做法少量尝试。
