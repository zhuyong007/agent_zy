# 媒体重命名器与确认对话框设计

## 目标

在现有照片重命名器基础上加入常见视频文件支持，并将执行时的浏览器原生确认框替换为站内主题化对话框。

## 方案

- 保留现有 `/api/tools/photo-renamer/*` API，避免破坏已有调用；页面文案升级为“照片和视频名称修改”。
- 照片继续读取 EXIF `DateTimeOriginal`、`CreateDate`，读取失败时回退文件修改时间。
- 视频支持 `mp4`、`mov`、`m4v`、`avi`、`mkv`、`webm`，优先通过本机 `ffprobe` 读取 `format.tags.creation_time`，失败或未安装时回退文件修改时间。
- 扫描、排序、冲突编号、状态校验、两阶段重命名、失败回滚与撤销逻辑由照片和视频共用。
- 共享类型增加 `video-metadata` 时间来源，保留既有 preview、execute、undo 数据结构。

## 确认对话框

- 点击“确认执行重命名”后打开页面内 modal，不立即执行。
- modal 展示待重命名文件数量、操作说明和可撤销提示。
- 主按钮执行重命名，次按钮取消；点击遮罩或按 `Escape` 取消。
- 执行中禁用关闭操作，避免状态中断。
- 对话框使用现有主题变量，兼容日间和夜间模式。

## 测试

- 服务测试覆盖视频扫描、视频元数据优先级和 mtime 回退。
- 页面测试覆盖 modal 打开、取消、确认执行，不再依赖 `window.confirm`。
- 完整运行测试、类型检查和 Web 构建。
