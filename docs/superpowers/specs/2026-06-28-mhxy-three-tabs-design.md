# 梦幻西游交易三 Tab 设计

## 目标

把现有“人民币主账本 / 召唤兽装备”两个工作区重组为三个职责清晰的 Tab：跨服交易记录、角色召唤兽及装备交易记录、物价记录。

## 页面分区

### 跨服交易记录

- 保留跨服交易的买入/卖出录入。
- 展示由跨服交易推导的当前库存及人民币估值。
- 展示跨服交易历史。
- 保留库存转移录入和库存转移记录。
- 不展示价格快照。

### 角色召唤兽及装备交易记录

- 只提供人民币买入和人民币卖出，不展示游戏币成本池、游戏币批次或购买币种切换。
- 当前库存仅包含状态为 `holding` 的资产。
- 交易历史仅包含状态为 `sold` 的资产。
- “库存价值”等于所有持有资产的人民币买入成本合计，即现有 `assetFlipSummary.holdingCostRmb`。
- “总盈亏”等于所有已售资产的人民币卖出价减买入价汇总，即现有 `assetFlipSummary.realizedProfitRmb`。
- 持有资产不计算浮动盈亏。

### 物价记录

- 独立展示价格快照录入表单。
- 独立展示价格快照历史及删除操作。

## 技术方案

沿用 `MhxyPage` 的现有查询、mutation 和样式体系，把工作区状态扩展为三个枚举值，并重新排列现有 JSX 区块。角色资产表单固定提交 `purchaseCurrency: "rmb"`，不修改后端模型，从而兼容已有数据；已有游戏币购入的历史资产仍按其冻结的 `buyPriceRmb` 作为人民币成本展示。

## 验证

- 组件测试验证三个 Tab 均存在且内容相互隔离。
- 组件测试验证角色当前库存只出现持有资产，交易历史只出现已售资产。
- 组件测试验证库存价值和总盈亏采用确认后的人民币口径。
- 组件测试验证角色资产表单不再提供游戏币维度并固定提交人民币购买方式。
- 运行页面组件测试、TypeScript 类型检查和 Web 构建。
