# 热点新闻子 Agent 设计

## 目标

构建一个“热点新闻”子 Agent：每 30 分钟从用户配置的信源增量拉取 AI、科技、经济、娱乐、国际等新闻，只把本轮新增内容交给归纳逻辑，聚合为可浏览的热点列表，并由用户主动触发单条新闻分析。

## 前端

- 页面路由使用已预留的 `/news`。
- 页面采用三栏情报工作台：左侧信源管理，中间分类热点列表，右侧选中新闻 inspector。
- 分类包括 `ai`、`technology`、`economy`、`entertainment`、`world`。
- 每条聚合新闻显示标题、摘要、分类、重要程度、信源数量、涉及信源、更新时间。
- 分析功能只在用户点击“分析”后触发，结果缓存并展示在 inspector 中。

## 后端

- `NewsState` 保存信源、原始新闻、聚合新闻、刷新时间、归纳时间和刷新状态。
- 信源可通过 API 添加，第一版支持 URL、名称、分类。
- 定时刷新间隔为 30 分钟。
- 每次刷新只处理 `lastFetchedAt` 之后的新内容，并基于链接/标题指纹去重。
- 聚合逻辑先使用本地 deterministic summarizer，保留替换为真实 LLM 的边界。
- 重要程度由信源覆盖数、分类和标题关键词推断，后续可替换为 LLM/embedding。

## API

- `GET /api/news` 返回新闻状态。
- `POST /api/news/sources` 添加信源。
- `POST /api/news/refresh` 手动触发增量刷新。
- `POST /api/news/items/:id/analyze` 分析指定聚合新闻。

## 成本边界

- 自动刷新只调用归纳逻辑，不调用深度分析。
- 已拉取并归纳过的历史原始新闻不会重复进入本轮归纳输入。
- 单条分析有缓存，重复点击同一条新闻返回已有分析。

## 验证

- 后端测试覆盖添加信源、增量刷新、聚类 sourceCount、分析缓存。
- 调度测试覆盖默认 30 分钟间隔。
- 前端通过 typecheck 和 web build 验证。
