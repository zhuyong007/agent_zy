# 王朝兴衰录重大事件主线设计

## 背景

历史知识模块的“朝代”模式包含“王朝兴衰录”“皇帝图鉴”“风云人物”“历史冷知识”四个模块。当前“王朝兴衰录”容易围绕多位人物展开，与“皇帝图鉴”和“风云人物”产生内容重复。

## 设计

仅调整 `history-agent` 的朝代模式生成提示词，不修改四件套的数据结构、前端展示或模块名称。

“王朝兴衰录”改为按时间顺序选择 5-8 个真正改变王朝走向的重大事件。每张图文卡片聚焦一个事件，讲清事件背景、过程、结果，以及它如何推动王朝建立、兴盛、转折、衰落或灭亡。

人物只作为事件参与者简要出现，仅说明其在事件中的作用。禁止把卡片写成人物生平、功绩盘点或帝王名单，并明确避免与“皇帝图鉴”和“风云人物”重复。

## 测试

更新 `agents/history-agent/src/index.test.ts` 的朝代模式测试，断言生成提示词包含：

- 重大事件主线与时间顺序要求。
- 5-8 个事件的数量要求。
- 每张卡片聚焦一个事件。
- 人物仅简要说明事件作用。
- 避免与“皇帝图鉴”和“风云人物”重复。

运行历史 Agent 聚焦测试与项目类型检查。
