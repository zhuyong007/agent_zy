# 梦幻西游交易模块对抗性审查修复设计

## 目标

修复已复现的账本正确性、同步数据完整性、时间口径、API 错误语义与页面回归问题，并在不改变现有模块边界的前提下降低行情估值的重复计算成本。

## 设计

1. 交易记账模式由本次请求的币种和显式兼容字段决定。人民币交易固定为 `directRmb`；游戏币交易默认使用钱包，只有导入的旧记录显式声明历史汇率模式时才保留 `legacyRate`。编辑导致钱包消耗变化时，不复用旧分配，统一由账本回放重新分配。
2. MHXY 同步导出前校验每类记录 ID 唯一。发现重复 ID 时中止同步并返回明确错误，禁止 `Map` 静默覆盖。数据仍保持现有键格式，避免协议迁移。
3. 资产 PATCH 使用 `null` 表示显式清除 `sellAt` 和 `sellPriceRmb`；两者同时为空时资产恢复持有状态，遗漏字段继续表示保持原值。
4. Dashboard 表示“截至当前时间”的账本状态。发生时间晚于服务时钟的交易、转移和提现保留在记录列表，但不参与当前库存、钱包与收益回放。服务接受可注入时钟，保证测试稳定。
5. API 将输入校验映射为 400、资源不存在映射为 404、账本冲突映射为 409、未知错误映射为 500。错误响应仍保持 `{ message }` 结构。
6. 行情估值先按区服和道具构建最新行情索引，再遍历库存计算，保持“无行情以成本兜底”的现有公式。

## 兼容性

- 旧 `legacyRate` 记录继续按旧口径回放。
- PATCH 中未传卖出字段时不改变资产状态；只有显式 `null` 才清除。
- 未来记录不会丢失，到达发生时间后自动进入 Dashboard 口径。
- 同步协议键名不变；仅把原本的数据丢失改为显式拒绝。

## 验证

每个问题先添加最小失败测试，确认失败原因后实现修复。最终运行 MHXY 服务、API、同步和页面测试，以及 `npm run typecheck`、`npm run build:web`。
