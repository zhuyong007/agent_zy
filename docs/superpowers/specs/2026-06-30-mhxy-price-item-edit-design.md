# 梦幻西游物价道具编辑设计

## 目标

用户查看某个道具的价格走势时，可以编辑该道具的名称和来源。编辑作用于当前走势下的全部历史快照，不改变任何价格、币种或采集时间。

当编辑后的“道具名 + 来源”已经存在时，系统必须先显示应用内二次确认弹窗；只有用户明确确认后，才把两条走势合并。

## 数据边界

物价观察列表中的一个“道具”不是独立实体，而是按“来源 + 道具名”聚合的一组 `MhxyPriceSnapshot`。因此编辑道具的本质是批量修改一组快照的聚合身份。

编辑允许修改：

- 道具名。
- 来源 / 区服。

编辑必须保留：

- 快照 ID。
- 人民币或游戏币价格字段。
- 历史兑换比例。
- 采集时间。
- 创建时间。
- 备注。

来源允许留空。空来源继续在界面显示为“未分类来源”，但不会把该展示占位写入真实数据。

## 交互设计

- 在当前道具趋势标题区增加“编辑道具”入口，与“记录新价格”并列。
- 点击后打开紧凑编辑表单，预填当前道具名和原始来源。
- 普通重命名直接保存，成功后刷新数据并保持新序列选中。
- 保存期间禁用提交按钮，防止重复操作。
- 保存失败时保持表单与输入，使用页面现有错误区域展示原因。
- 取消编辑时关闭表单，不修改数据。

如果目标序列已经存在：

- 不立即提交。
- 显示应用内确认弹窗，不使用 `window.confirm`。
- 弹窗明确显示当前序列将移动的历史记录数量、目标序列已有记录数量，以及合并后的道具名和来源。
- 点击“取消”关闭弹窗并返回编辑表单，输入保持不变。
- 点击“确定合并”后，携带明确确认标记提交。

## API 与服务设计

新增共享输入类型 `MhxyPriceSeriesUpdateInput`：

```ts
interface MhxyPriceSeriesIdentity {
  itemName: string;
  serverName?: string;
}

interface MhxyPriceSeriesUpdateInput {
  current: MhxyPriceSeriesIdentity;
  next: MhxyPriceSeriesIdentity;
  confirmMerge?: boolean;
}
```

新增接口：

```text
PATCH /api/mhxy/price-series
```

接口返回更新后的快照列表，以及 `updatedCount`、`merged` 和目标序列合并前的 `targetRecordCount`，供前端更新状态和展示结果。

服务层在一次写入中完成：

1. 规范化当前身份和目标身份：道具名去除首尾空格且不能为空，空白来源归一为缺失值。
2. 找出当前序列的全部快照；不存在时返回明确错误。
3. 判断目标身份是否与当前身份相同；相同时直接返回当前序列，不产生无意义写入。
4. 判断目标序列是否已经存在。
5. 如果目标存在且 `confirmMerge` 不是 `true`，拒绝写入并返回“需要确认合并”的明确错误。
6. 修改当前序列全部快照的 `itemName`、可选 `serverName` 和 `updatedAt`，其余字段保持不变。
7. 一次性写回完整快照数组并返回更新后的记录和合并信息。

后端始终执行合并确认校验，不能只依赖前端预判。这样即使请求被直接调用，或编辑期间目标序列刚被其他同步操作创建，也不会未经确认合并。

## 前端状态与数据流

`PriceTrendWorkspace` 已掌握当前序列的原始 `itemName/serverName`、展示来源和记录数量。它负责：

- 打开编辑表单并提供预填值。
- 根据当前快照列表预判目标序列是否存在，以便在发请求前展示确认弹窗。
- 普通保存时发送 `confirmMerge: false`。
- 用户确认合并时发送同一目标并设置 `confirmMerge: true`。
- 保存成功后记录目标序列键、刷新 `mhxy` 查询，并优先选中新键对应序列。

如果前端未预判到冲突但后端拒绝合并，界面保留表单并显示明确错误；服务端不写入任何数据。前端刷新价格列表后，用户再次保存时会基于最新目标序列展示确认弹窗。

编辑状态按当前序列键隔离。切换观察道具时关闭编辑表单和确认弹窗，避免把旧序列的输入提交到新序列。

## 错误与边界处理

- 道具名为空：前后端同时阻止提交。
- 来源只包含空格：保存为无来源，不保存展示占位。
- 当前序列不存在：不写入，返回“价格序列不存在”。
- 目标与当前完全相同：不修改记录，关闭编辑表单并保持当前选择。
- 目标已存在但未确认：不写入任何记录。
- 确认合并：保留两组所有历史快照，由现有聚合和排序逻辑展示为一条走势。
- 请求失败：不关闭编辑表单，不清空输入。
- 数据刷新或同步导致当前序列变化：按实际序列键重建编辑状态。

## 视觉与无障碍

- “编辑道具”使用次级按钮层级，不抢占“记录新价格”的主要操作。
- 编辑表单沿用现有行情终端色彩和控件样式。
- 确认弹窗使用 `role="dialog"`、可读标题和明确的“取消 / 确定合并”按钮。
- 窄屏下表单和弹窗保持单列，不产生横向滚动。
- 键盘焦点样式与现有物价模块一致。

## 测试与验收

共享校验与服务测试覆盖：

- 批量重命名保留快照业务字段和创建时间。
- 空来源不会写入“未分类来源”。
- 当前序列不存在时拒绝写入。
- 目标已存在且未确认时不写入。
- 确认后合并并保留两组全部记录。
- 目标与当前相同时不产生无意义更新。

API 测试覆盖：

- `PATCH /api/mhxy/price-series` 的正常更新。
- 空道具名校验。
- 未确认合并拒绝和确认合并成功。

前端测试覆盖：

- 当前道具显示编辑入口并预填名称、来源。
- 普通编辑提交后保持目标走势选中。
- 检测到目标已存在时不立即请求并显示确认弹窗。
- 取消弹窗不修改数据且保留编辑输入。
- 确认后携带 `confirmMerge: true` 提交。
- 切换道具时关闭编辑和确认状态。
- API 失败时保留输入。
- 空来源编辑不会提交展示占位。

验收标准：用户可以在当前价格走势中修改道具名称或来源；如果新身份已存在，系统在任何数据写入前要求二次确认，确认后两组历史完整合并。
