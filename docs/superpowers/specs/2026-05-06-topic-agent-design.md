# AI 自媒体选题 Agent 设计

## 目标

新增一个“AI 自媒体选题”子 Agent：每 3 小时自动生成一批适合做 AI 自媒体的选题，在首页展示当前推送，并在独立路由中查看历史选题。

## 选题策略

- 第一版优先读取现有 `news.items`，将热点新闻转化为内容选题。
- 评分维度包括 AI 相关性、时效性、受众收益、反差/争议度、可落地讲述角度和素材可信度。
- 当热点不足时，用常青型 AI 自媒体主题补位，保证每次推送仍有可用选题。
- 每条选题包含标题、钩子、推荐理由、内容角度、目标受众、为什么现在做、来源新闻和分数。

## 后端

- 新增 `TopicState` 保存当前推送、历史选题、生成时间、下次运行时间、状态和错误。
- 新增 `topic-agent`，支持 `generate` 动作。
- 调度器新增 3 小时间隔，启动时立即跑一次，之后按间隔刷新。
- 控制面暴露 `GET /api/topics` 和 `POST /api/topics/generate`。
- dashboard 返回 `topics`，供首页直接展示最新推送。

## 前端

- 首页增加“AI 自媒体选题”模块，展示最新 3 条选题、分数、钩子和更新时间。
- 新增 `/topics` 路由，展示当前推送、历史选题、来源热点和手动生成入口。
- 顶部命令路由增加“选题”入口。

## 验证

- Agent 测试覆盖从热点生成选题、无热点补位、历史保留和去重。
- 控制面测试覆盖 topic API 和 3 小时调度默认值。
- 运行 typecheck 和 web build 验证前端类型与构建。
